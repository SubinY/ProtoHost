为何选 Koa 而非 Next.js

需求KoaNext.js API
200MB multipart✓ multer 成熟需额外配置
HTML/JS 内容改写✓ 完全可控✗ 不适合
流式 ZIP 下载✓ ReadStream可行但别扭
纯 API 无 SSR✓ 轻量框架偏重
Vue 分离部署✓ 天然匹配需刻意分离

Next.js 适合同仓全栈 + RSC。ProtoHost 核心是文件服务与 Axure 适配，Koa 更合适。

认证请求链路

Client  POST /api/auth/login
  → Nginx proxy
  → Koa Router
  → Zod safeParse(body)
  → auth.service.login()
      → drizzle: users.findByEmail
      → bcrypt.compare
      → jose.sign({ sub: userId })
  → { token, email }

私密分享预览链路

ViewPage  POST /api/share/:slug/verify
  → share.service.verifyPassword()
  → jose.sign({ type:'view', slug })  viewToken 2h

iframe  GET /files/start.html?viewToken=...
  → share.routes.serveFile()
  → 路径 canonical 校验
  → 若 .html → axure-injector 注入 script
  → 若 axplayer.js / doc.js → 字符串替换
  → 其他资源直接 ReadStream

上传请求链路

UploadPage  POST /api/projects/upload  (multipart 200MB)
  → @koa/multer 临时文件 / 流
  → auth 中间件 → userId
  → upload.service:
      1. lib/zip.extract → uploads/{userId}/{ts}/
      2. 检测 entryFile (index.html / start.html)
      3. db.transaction:
         - insert/update projects
         - insert project_versions
      4. 失败 → 删除目录 rollback
  → ProjectDTO JSON
